# Spec-Driven Development — Do Código ao Contrato na Era dos AI Coding Assistants

**Tema:** metodologia
**Fonte:** https://arxiv.org/html/2602.00180v1

## Resumo

- Paper acadêmico (arxiv) que formaliza SDD como paradigma de engenharia de software para a era de assistentes de código IA
- Define formalmente que em SDD a especificação é o artefato primário e o código é o output gerado — inversão do modelo tradicional
- Analisa como specs reduzem o espaço de busca do modelo de linguagem, resultando em código mais consistente e auditável
- Propõe métricas para medir aderência entre spec e código gerado — o que falta em ferramentas atuais
- Identifica como principal limitação atual a ausência de validação automática de specs antes da geração

## Por que isso importa

Ter um paper no arxiv sobre SDD sinaliza que a prática está sendo formalizada academicamente — não é apenas hype de produto. As métricas propostas ajudam a entender quando uma spec é boa o suficiente para um agente executar autonomamente.

## Citação

> "The specification becomes the primary artifact; the application logic, infrastructure configuration, and event schemas all derive from it."
